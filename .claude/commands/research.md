# Stock Research Command

指定された銘柄について追加リサーチを行い、システムの判断と最新情報を比較します。

## 引数

- `$ARGUMENTS`: 銘柄シンボル（例: AAPL, TSLA, 7203.T）

## 実行手順

### 1. システムの現在の判断を確認

まず、Supabaseデータベースから銘柄の現在の判断を取得します。

```bash
python scripts/research_stock.py $ARGUMENTS
```

このスクリプトが返す情報：
- システムの判断（BUY/HOLD/AVOID）
- 信頼度
- キーファクター
- 識別されたリスク
- ルールベーススコア

### 2. 最新ニュースを検索

WebSearchツールを使って以下を検索：

**検索クエリ例：**
- `$ARGUMENTS stock news today`
- `$ARGUMENTS 株価 ニュース 最新` （日本株の場合）
- `$ARGUMENTS earnings analyst rating`

### 3. 比較分析

以下の観点で分析：

1. **センチメント整合性**: 最新ニュースの雰囲気はシステム判断と一致しているか？
2. **新規材料**: バッチ実行後に出た重要なニュースはあるか？
3. **リスク更新**: 新たに識別すべきリスクはあるか？
4. **判断変更の必要性**: システムの判断を見直すべき材料があるか？

## 日本株の場合

日本株（シンボルが `.T` で終わる場合や4桁数字の場合）は：
- 日本語でのニュース検索も実施
- Yahoo!ファイナンスJapan等の情報も参照
- 決算発表スケジュールの確認

## 注意事項

- このコマンドはリアルタイムの株価取得は行わない（API制限のため）
- ニュース検索結果は参考情報であり、投資判断は自己責任
- バッチ実行からの時間経過を考慮すること
